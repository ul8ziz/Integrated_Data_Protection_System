# كيان المستخدمين — العناصر والسمات
# Users Entity — Elements and Attributes

## نظرة عامة

هذا المستند يحدد كيان المستخدمين (Users) وعناصره وسماته بناءً على النموذج الفعلي في المشروع (`backend/app/models_mongo/users.py`).

---

## الكيان المركزي

**المستخدمين (Users)** — يمثل حسابات المستخدمين في نظام حماية البيانات.

---

## السمات (Attributes)

| السمة | الاسم بالإنجليزية | النوع | مطلوب | الوصف |
|------|-------------------|-------|-------|-------|
| المعرف الفريد | id | ObjectId (string) | نعم | معرف فريد يُنشأ تلقائياً من MongoDB |
| اسم المستخدم | username | string | نعم | 3–100 حرف |
| البريد الإلكتروني | email | EmailStr | نعم | بريد صالح |
| كلمة المرور | hashed_password | string | نعم | مشفرة (bcrypt) — لا تُخزَّن كنص صريح |
| دور المستخدم | role | UserRole | نعم | admin, regular, manager |
| حالة المستخدم | status | UserStatus | نعم | pending, approved, rejected, active |
| حالة التفعيل | is_active | boolean | نعم | false حتى الموافقة |
| القسم | department_id | string (ObjectId) | لا | مرجع إلى Department |
| تاريخ الموافقة | approved_at | datetime | لا | وقت موافقة المدير |
| معرف المستخدم الذي وافق | approved_by | string | لا | User ID للموافق |
| سبب الرفض | rejection_reason | string | لا | عند status=rejected |
| تاريخ الإنشاء | created_at | datetime | نعم | وقت إنشاء الحساب |
| آخر تسجيل دخول | last_login | datetime | لا | آخر دخول ناجح |

---

## دور المستخدم (User Role)

| القيمة | العربية | الوصف |
|-------|---------|-------|
| `regular` | مستخدم عادي | تحليل نصوص/ملفات، مراقبة البريد |
| `admin` | مدير النظام | إدارة المستخدمين، السياسات، التقارير، التنبيهات |
| `manager` | مدير قسم | إدارة مستخدمي القسم نفسه فقط |

**تصحيح المخطط الأصلي:** كان يعرض `admin` و`regular` فقط — المشروع يضيف `manager`.

---

## حالة المستخدم (User Status)

| القيمة | العربية | الوصف |
|-------|---------|-------|
| `pending` | في انتظار الموافقة | طلب تسجيل جديد |
| `approved` | موافق عليه | تمت الموافقة، لم يُفعّل بعد |
| `rejected` | مرفوض | رُفض الطلب |
| `active` | نشط | موافق عليه ومُفعّل — يمكن تسجيل الدخول |

**تصحيح المخطط الأصلي:** كان يعرض "معتمد نشط" و"مرفوض غير نشط" فقط — المشروع يستخدم 4 حالات.

---

## حالة التفعيل (is_active)

- **false:** الحساب غير مفعّل (افتراضي حتى الموافقة)
- **true:** الحساب مفعّل ويمكن تسجيل الدخول (مع status=approved أو active)

---

## سمات غير مخزنة في قاعدة البيانات

| السمة | الاستخدام |
|-------|-----------|
| تأكيد كلمة المرور (Confirm Password) | التحقق من النموذج عند التسجيل فقط — لا يُخزَّن |
| كلمة المرور (Password) | يُخزَّن فقط كـ hashed_password |
